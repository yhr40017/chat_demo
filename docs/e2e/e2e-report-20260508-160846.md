## 발견 사항

### 1. RAG 한글 토큰화 제약 (중요도: Medium)

**현상**: 한글 쿼리("머신러닝 딥러닝 인공지능")가 한글 문서("인공지능과 머신러닝에 대한 내용입니다")와 매칭되지 않음 (score=0.0).

**원인**: `vector_store.py:15`의 `_tokenize()` 함수가 `re.findall(r"[\w]+", text.lower())`를 사용하여 단순 단어 분리만 수행. 한국어 조사("과", "에", "은", "는" 등)가 단어에 붙어 있어 "인공지능" ≠ "인공지능과"로 불일치.

**영향**: 한글 문서에 대한 RAG 검색 성능이 크게 저하됨. 영문 문서는 정상 동작.

**권장 조치**: 한국어 형태소 분석기(예: `konlpy`, `mecab`, `kiwi`) 도입 또는 n-gram 기반 토큰화로 변경. 최소한 조사 제거(`re.sub(r'[은는이가을를에서의와과로]$', '', token)`) 적용.

**관련 코드**: `backend/app/vector_store.py:15-16`

### 2. 빈 메시지 API 수준 검증 부재 (중요도: Low)

**현상**: 프론트엔드에서 `!input.trim()` 가드로 빈 메시지 전송을 차단하지만, API 수준에서는 공백 메시지(" ")도 정상 처리되어 Ollama에 전달됨.

**영향**: API 직접 호출 시 빈 메시지가 LLM에 전달되어 불필요한 리소스 사용.

**권장 조치**: `ChatRequest` 스키마에 message 필드 validation 추가 (`min_length=1` + `strip_whitespace=True` 또는 커스텀 validator).

**관련 코드**: `backend/app/schemas.py` — ChatRequest, `frontend/src/components/ChatInput.tsx:28`

### 3. 긴 메시지(10,000자) 시 assistant 응답 내용 비어있음 (중요도: Low)

**현상**: 10,000자 반복 문자("가"×10000) 전송 시 SSE done 이벤트는 정상이나, DB에 저장된 assistant 메시지 content 길이가 0.

**영향**: 극단적 입력에서 LLM 응답이 비어있을 수 있음. 사용자 시나리오에서는 드문 케이스.

**관련 코드**: `backend/app/routes/chat.py:218-222`
