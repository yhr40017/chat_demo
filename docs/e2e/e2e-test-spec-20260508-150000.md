## 1. 테스트 환경 및 실행 가이드

### 1.1 사전 요구사항

| 구성요소 | 버전 | 용도 |

| Python | 3.10+ | Backend 런타임 |

### 1.2 환경 설정

```bash
# 1. MySQL 컨테이너 기동
docker exec mysql84 mysql -uroot -pMobigen_1234 -e "CREATE DATABASE IF NOT EXISTS chat_demo CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 2. Backend 기동
cd backend && source venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# 3. Frontend 기동
cd frontend && npm start  # port 3000

# 4. Ollama 모델 확인
ollama list  # gemma4:26b 존재 확인
```

### 1.3 테스트 실행 순서 (권장)

1. **CONV** (대화 관리) — 기본 CRUD, 다른 영역의 전제
2. **FILE** (파일 첨부) — CONV 의존
3. **KNOW** (지식 문서) — 독립 실행 가능, RAG의 전제
4. **RAG** (RAG 파이프라인) — KNOW 의존 (문서 업로드 필요)
5. **CHAT** (채팅 & 스트리밍) — CONV 의존, Ollama 필요
6. **SPMT** (시스템 프롬프트) — CONV 의존
7. **SRCH** (검색) — CONV + CHAT 의존 (메시지 존재 필요)
8. **EXIM** (내보내기/가져오기) — CONV + CHAT 의존
9. **UICF** (모델/테마) — 독립 실행 가능
