## 1.2 시스템 목적

Ollama 로컬 LLM을 활용한 웹 기반 채팅 애플리케이션으로, 사용자가 로컬에 설치된 다양한 LLM 모델과 대화하고 문서를 참조하여 지식 기반 응답을 받을 수 있는 시스템입니다.

### 주요 목표

- Ollama 로컬 LLM을 활용한 실시간 스트리밍 채팅 인터페이스 제공
- 멀티 세션 대화 관리 (생성, 삭제, 전환, 이력 DB 저장)
- 문서 업로드 기반 RAG(Retrieval-Augmented Generation) 지식 저장소 운영
- 대화별 파일 첨부 및 컨텍스트 활용
- 시스템 프롬프트 커스터마이징을 통한 AI 역할 설정

## 1.3 시스템 범위

### 관리/처리 대상

| 대상 항목 | 유형 | 비고 |
|-----------|------|------|
| 대화(Conversation) | 데이터 | 멀티 세션 대화방 생성/삭제/전환 |
| 메시지(Message) | 데이터 | 사용자/AI 메시지 이력 저장 |
| 첨부파일(Attachment) | 파일/데이터 | 대화별 문서 첨부, 텍스트 추출 후 컨텍스트 활용 |
| 지식 문서(KnowledgeDocument) | 파일/데이터 | 글로벌 RAG 지식 저장소, 문서 요약 및 청크 인덱싱 |
| LLM 모델 | 외부 서비스 | Ollama에 설치된 모델 목록 조회 및 선택 |

### 기능 범위

| 기능 영역 | 약어 | 설명 |
|-----------|------|------|
| 대화 관리 | CONV | 대화방 CRUD, 검색, 내보내기/가져오기 |
| 채팅 | CHAT | SSE 스트리밍 대화, 자동 제목 생성, RAG 통합 |
| 모델 관리 | MODEL | Ollama 모델 목록 조회 및 전환 |
| 파일 첨부 | ATTACH | 대화별 파일 업로드/삭제, 텍스트 추출 |
| 지식 저장소 | KNOW | 문서 업로드, 요약 생성, 청크 분할, 하이브리드 검색 |
| 시스템 프롬프트 | SYSPROMPT | AI 역할 커스터마이징, 프리셋 제공, DB 영속 |
| UI/테마 | UI | 다크/라이트 테마 전환, 마크다운 렌더링, 코드 하이라이팅 |
