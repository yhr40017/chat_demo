# 3. 모듈별 상세 기능

## 3.1 백엔드 — API 서버

### 3.1.1 FastAPI 엔트리포인트 (main.py)

> **Main File:** `backend/app/main.py`
> **포트:** 8000 | **Config Name:** `app.main:app`

#### 기능 개요

FastAPI 애플리케이션 초기화, CORS 설정, 라우터 등록, 데이터베이스 테이블 자동 생성을 담당합니다.

#### 처리 흐름

```
앱 시작 (lifespan)
  → SQLAlchemy Base.metadata.create_all (테이블 자동 생성)
  → 라우터 5개 등록 (conversations, chat, models, attachments, knowledge)
  → CORS 미들웨어 설정 (origin: localhost:3000)
  → /api/health 헬스체크 엔드포인트

앱 종료
  → DB 엔진 dispose
```

#### 주요 클래스/파일

| 파일 | 역할 |
|------|------|
| `main.py` | FastAPI 앱 생성, lifespan 관리, 라우터/미들웨어 등록 |
| `config.py` | pydantic-settings 기반 환경설정 (DATABASE_URL, OLLAMA_BASE_URL) |
| `database.py` | SQLAlchemy async 엔진/세션 팩토리, get_db 의존성 |

## 3.2 백엔드 — 대화 관리

### 3.2.1 대화 라우터 (conversations.py)

> **Main File:** `backend/app/routes/conversations.py`
> **Prefix:** `/api/conversations`

#### 기능 개요

대화방 CRUD 기능과 대화 검색, 내보내기(JSON/Markdown), 가져오기(JSON) 기능을 제공합니다.

#### 처리 흐름

```
대화 검색 (GET /search?q=)
  → Message + Conversation JOIN
  → content LIKE 또는 title LIKE 매칭
  → 매칭 위치 기준 ±40자 snippet 생성
  → 최대 50건 반환

대화 내보내기 (GET /{id}/export?format=)
  → 대화 + 메시지 로드
  → format=json: JSON 직렬화 + UTF-8 파일명 (URL-encode)
  → format=markdown: 마크다운 포맷 변환 + UTF-8 파일명

대화 가져오기 (POST /import)
  → ConversationImport 스키마 검증
  → Conversation 생성 → Message 일괄 삽입
```

#### 주요 클래스/파일

| 파일/클래스 | 역할 |
|-------------|------|
| `routes/conversations.py` | 대화 CRUD, 검색, 내보내기/가져오기 엔드포인트 |
| `schemas.py::ConversationCreate` | 대화 생성 요청 (title, model, system_prompt) |
| `schemas.py::ConversationUpdate` | 대화 수정 요청 (title, model, system_prompt) |
| `schemas.py::SearchResult` | 검색 결과 응답 |
| `schemas.py::ConversationImport` | 가져오기 요청 |

### 3.2.2 채팅 라우터 (chat.py)

> **Main File:** `backend/app/routes/chat.py`
> **Prefix:** `/api/conversations/{id}/chat`

#### 기능 개요

SSE 스트리밍 채팅 처리, 2-stage RAG 통합, 자동 제목 생성을 담당하는 핵심 모듈입니다.

#### 처리 흐름

```
채팅 요청 (POST /{id}/chat)
  → 대화 조회 (messages + attachments 선 로딩)
  → 사용자 메시지 DB 저장
  ──── RAG 파이프라인 시작 ────
  → [1단계] KnowledgeDocument 요약 로드 (status=ready, summary!=null)
  → [1단계] TF-IDF 코사인 유사도로 질문-요약 매칭 (threshold: 0.01)
  → [2단계] vector_store.search() 하이브리드 청크 검색 (최대 10건)
  → 우선 문서 청크 앞 배치 + 비우선 문서 뒤 배치 (최대 7건)
  → 우선 문서 요약을 컨텍스트에 추가
  ──── RAG 파이프라인 종료 ────
  → 첨부파일 컨텍스트 구성 (첨부 content_text[:8000])
  → 시스템 프롬프트 조합: 사용자 설정 + RAG + 첨부
  → 대화 이력 messages 배열 구성
  → Ollama /api/chat 스트리밍 호출 (timeout: 300s)
  → SSE로 thinking/content 토큰 전송
  → 응답 완료:
    → AI 메시지 DB 저장 (references 포함)
    → 첫 메시지인 경우 자동 제목 생성 (Ollama /api/generate)
    → done 이벤트 전송 (title + references)
```

#### 핵심 함수

| 함수 | 역할 |
|------|------|
| `chat()` | 메인 채팅 엔드포인트 (SSE StreamingResponse) |
| `generate()` | SSE 스트리밍 제너레이터 (비동기) |
| `generate_title()` | 첫 대화 기반 자동 제목 생성 (15자 이내 한국어) |
| `_compute_summary_similarity()` | TF-IDF 코사인 유사도 기반 문서 요약 매칭 |

## 3.3 백엔드 — 지식 저장소

### 3.3.1 지식 라우터 (knowledge.py)

> **Main File:** `backend/app/routes/knowledge.py`
> **Prefix:** `/api/knowledge`

#### 기능 개요

지식 문서 업로드, 백그라운드 처리(텍스트 추출 → LLM 요약 → 청크 분할 → 인덱싱), 상태 조회, 삭제 기능을 제공합니다.

#### 처리 흐름

```
문서 업로드 (POST /upload)
  → 파일 크기 검증 (최대 20MB)
  → KnowledgeDocument 레코드 생성 (status: processing)
  → BackgroundTasks에 process_document 등록
  → 즉시 응답 반환

백그라운드 처리 (process_document)
  → file_parser.extract_text() 호출
  → 텍스트가 비어있으면 에러 상태로 전환
  → generate_summary(): Ollama /api/generate로 3~5문장 요약 생성
    → 모델: gemma4:26b, 입력: 텍스트 앞 4000자
  → chunker.split_text(): 500자 청크 분할 (50자 오버랩)
  → vector_store.add_chunks(): BM25+TF-IDF 인덱스에 추가
  → 상태 업데이트 (status: ready, chunk_count, summary)

문서 삭제 (DELETE /{id})
  → vector_store.delete_by_doc_id() 인덱스에서 제거
  → DB 레코드 삭제
```

#### 주요 클래스/파일

| 파일/함수 | 역할 |
|-----------|------|
| `routes/knowledge.py` | 지식 문서 CRUD 엔드포인트 |
| `generate_summary()` | LLM 기반 문서 요약 생성 (gemma4:26b, timeout: 120s) |
| `process_document()` | 백그라운드 문서 처리 파이프라인 |

### 3.3.2 검색 엔진 (vector_store.py)

> **Main File:** `backend/app/vector_store.py`
> **영속 파일:** `backend/knowledge_data/index.pkl`

#### 기능 개요

BM25 + TF-IDF 하이브리드 검색 엔진으로, 문서 청크를 인덱싱하고 질문에 대한 관련 청크를 검색합니다.

#### 처리 흐름

```
인덱스 빌드 (_rebuild_index)
  → 모든 청크를 토큰화 (정규식: [\w]+)
  → BM25Okapi 인덱스 빌드
  → TfidfVectorizer fit_transform

검색 (search)
  → BM25 스코어 계산 → 정규화 (max 기준)
  → TF-IDF 코사인 유사도 계산
  → 하이브리드 점수 = BM25 × 0.5 + TF-IDF × 0.5
  → 점수 상위 n건 반환 (threshold: 0.05)

영속화
  → chunks + doc_ids를 pickle로 저장/로드
  → 앱 시작 시 자동 로드, 변경 시 자동 저장
```

#### 주요 클래스/파일

| 클래스/함수 | 역할 |
|-------------|------|
| `HybridSearch` | 싱글톤 검색 엔진 클래스 |
| `add_chunks()` | 문서 청크 인덱스 추가 |
| `search()` | BM25+TF-IDF 하이브리드 검색 |
| `delete_by_doc_id()` | 문서 ID 기준 인덱스 삭제 |
