## 2.2 계층별 역할

| 계층 | 역할 | 주요 구성 |
|------|------|-----------|
| 프레젠테이션 | 사용자 인터페이스, 상태 관리, API 호출 | React 컴포넌트 (App.tsx + 7개 하위 컴포넌트) |
| API 게이트웨이 | REST 엔드포인트, SSE 스트리밍, CORS | FastAPI 라우터 5개 (conversations, chat, models, attachments, knowledge) |
| 비즈니스 로직 | 채팅 처리, RAG 검색, 문서 파싱, 요약 생성 | chat.py (2-stage RAG), knowledge.py (LLM 요약), file_parser.py, chunker.py |
| 데이터 접근 | ORM 기반 CRUD, 비동기 DB 세션 | SQLAlchemy async + aiomysql |
| 검색 엔진 | 청크 인덱싱, 하이브리드 검색 | vector_store.py (BM25 + TF-IDF), pickle 파일 영속 |
| 외부 시스템 | LLM 추론, 제목 생성, 요약 생성 | Ollama REST API (/api/chat, /api/generate, /api/tags) |
| 영속 계층 | 대화/메시지/첨부/지식문서 저장 | MySQL 8.4 (Docker), knowledge_data/index.pkl |

## 2.4 통신 흐름 요약

> 다이어그램 원본: [`diagrams/network-flow.puml`](diagrams/network-flow.puml)

### 채팅 요청 흐름 (RAG 포함)

```
1. [프론트엔드] POST /api/conversations/{id}/chat  { message: "..." }
2. [chat.py] 대화 + 메시지 이력 로드 (MySQL)
3. [chat.py] 사용자 메시지 DB 저장
4. [chat.py] 1단계 RAG: 지식문서 요약 로드 → TF-IDF 코사인 유사도 계산
5. [chat.py] 2단계 RAG: vector_store.search() → BM25+TF-IDF 하이브리드 청크 검색
6. [chat.py] 우선 문서 청크 재정렬 + 요약 컨텍스트 추가
7. [chat.py] 시스템 프롬프트 구성 (사용자 설정 + RAG 컨텍스트 + 첨부 컨텍스트)
8. [chat.py] Ollama /api/chat 스트리밍 호출
9. [chat.py → 프론트엔드] SSE 토큰 스트리밍 (thinking + content)
10. [chat.py] 응답 완료 → AI 메시지 DB 저장 + 참조 정보 저장
11. [chat.py] 첫 메시지인 경우: Ollama /api/generate로 자동 제목 생성
12. [chat.py → 프론트엔드] SSE done 이벤트 (title + references)
```

### 지식 문서 처리 흐름

```
1. [프론트엔드] POST /api/knowledge/upload  (multipart/form-data)
2. [knowledge.py] KnowledgeDocument 레코드 생성 (status: processing)
3. [knowledge.py] 백그라운드 태스크 등록
4. [BackgroundTask] 텍스트 추출 (file_parser.py)
5. [BackgroundTask] LLM 요약 생성 (Ollama /api/generate, gemma4:26b)
6. [BackgroundTask] 청크 분할 (chunker.py, 500자/50자 오버랩)
7. [BackgroundTask] BM25+TF-IDF 인덱스 추가 (vector_store.py → index.pkl)
8. [BackgroundTask] 상태 업데이트 (status: ready, chunk_count, summary)
```

## 2.5 외부 연동 시스템

| 연동 대상 | 프로토콜 | 포트 | 연동 모듈 | 설명 |
|-----------|----------|------|-----------|------|
| Ollama LLM Server | HTTP REST | 11434 | chat.py, knowledge.py, models.py | 채팅 스트리밍, 제목 생성, 요약 생성, 모델 목록 |
| MySQL 8.4 | TCP (aiomysql) | 3306 | 전체 라우터 (SQLAlchemy) | 대화, 메시지, 첨부파일, 지식문서 영속 |

### Ollama API 사용 엔드포인트

| Ollama API | 사용처 | 용도 |
|------------|--------|------|
| `POST /api/chat` | chat.py | 대화 스트리밍 (stream: true) |
| `POST /api/generate` | chat.py | 자동 제목 생성 (stream: false) |
| `POST /api/generate` | knowledge.py | 문서 요약 생성 (stream: false, model: gemma4:26b) |
| `GET /api/tags` | models.py | 설치된 모델 목록 조회 |

## 2.6 인프라 구성요소

| 구성요소 | 용도 | 접근 모듈 |
