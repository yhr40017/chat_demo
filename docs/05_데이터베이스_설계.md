# 5. 데이터베이스 설계

> ERD 다이어그램 원본: [`diagrams/db-erd.puml`](diagrams/db-erd.puml)

## 5.1 데이터베이스 개요

| 항목 | 내용 |
|------|------|
| DBMS | MySQL 8.4 (Docker) |
| 데이터베이스명 | chat_demo |
| 문자셋 | utf8mb4 (unicode_ci) |
| Driver | aiomysql |
| ORM | SQLAlchemy 2.0 (async, Mapped 타입) |
| Connection Pool | SQLAlchemy create_async_engine (기본 설정) |
| 세션 관리 | async_sessionmaker (expire_on_commit=False) |
| 테이블 생성 | Base.metadata.create_all (앱 시작 시 자동) |

### 접속 설정 (config.py)

```
DATABASE_URL = mysql+aiomysql://root:{password}@localhost:3306/chat_demo
```

## 5.6 테이블 관계 요약

### 주요 참조 관계

```
conversations (1) ──────┤├── messages (N)
     │                         conversation_id FK (CASCADE)
     │
conversations (1) ──────┤├── attachments (N)
                               conversation_id FK (CASCADE)

knowledge_documents (독립)
     └── vector_store index.pkl 과 doc_id로 연결 (애플리케이션 레벨)
```

### 관계 상세

| 부모 테이블 | 자식 테이블 | 관계 | FK | ON DELETE |
|-------------|-------------|------|-----|-----------|
| conversations | messages | 1:N | conversation_id | CASCADE |
| conversations | attachments | 1:N | conversation_id | CASCADE |

### 데이터 생명주기

```
[대화 생성]
  conversations 레코드 생성
    ↓
[메시지 교환]
  messages 레코드 누적 (user → assistant 교대)
    ↓
[파일 첨부] (선택적)
  attachments 레코드 생성 (텍스트 추출 후 저장)
    ↓
[대화 삭제]
  conversations 삭제 → messages, attachments CASCADE 삭제

[지식 문서 업로드]
  knowledge_documents 레코드 생성 (status: processing)
    ↓
  [백그라운드] 텍스트 추출 → 요약 생성 → 청크 분할 → 인덱스 추가
    ↓
  knowledge_documents 업데이트 (status: ready, chunk_count, summary)

[지식 문서 삭제]
  knowledge_documents 레코드 삭제 + vector_store 인덱스 제거
```

## 5.7 ORM 모델 매핑

> **Main File:** `backend/app/models.py`

| ORM 클래스 | 테이블 | relationship |
|------------|--------|-------------|
| `Conversation` | conversations | messages (1:N), attachments (1:N) |
| `Message` | messages | conversation (N:1) |
| `Attachment` | attachments | conversation (N:1) |
| `KnowledgeDocument` | knowledge_documents | - (독립) |

### ORM 특이사항

- `Mapped` 타입 기반 선언적 매핑 (SQLAlchemy 2.0 스타일)
- `expire_on_commit=False` 설정으로 커밋 후 객체 접근 가능
- `selectinload`로 N+1 문제 방지 (conversations → messages, attachments)
- `server_default=func.now()` 사용 (DB 서버 시간 기준)
- messages의 `references` 컬럼은 `JSON` 타입 (MySQL 8.x 네이티브 JSON)
